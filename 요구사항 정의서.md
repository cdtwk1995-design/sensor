[요구사항 정의서] 실시간 센서 모니터링 PWA
1. 개요
목적: Supabase sensor_logs 테이블의 데이터를 실시간으로 모니터링하고, 위험 수준별로 데이터를 시각적으로 분류하여 표시.

대상: 누구나 접근 가능한 웹 기반 대시보드(PWA).

2. 기능 요구사항 (Functional Requirements)
데이터 실시간 구독: Supabase Realtime SDK를 통해 sensor_logs 테이블의 신규 데이터를 실시간 수신 및 갱신.

위험 수준별 분류 표시:

전체 로그(검은색 영역): 수신되는 모든 센서 데이터 표시.

경고(주황색 영역): risk_level이 1인 데이터만 필터링하여 표시.

위험(빨간색 영역): risk_level이 2인 데이터만 필터링하여 표시.

위험 알림: risk_level이 2인 데이터 수신 시, 브라우저 오디오 API를 사용하여 즉시 경고음 발생.

PWA 지원: manifest.json 및 service-worker.js를 통해 모바일/데스크톱 설치 및 오프라인 접근성 확보.

3. 디자인 및 UI/UX 개선 요구사항 (UI Improvement)
사용자 가독성과 즉각적인 상황 인지를 위해 다음 디자인 규칙을 적용합니다.

데이터 카드(Card) 구조:

텍스트의 단순 나열이 아닌, 개별 데이터를 타임스탬프와 값이 정렬된 '카드' 형태로 생성하여 정보의 구분 명확화.

위험 상태 시각적 강조:

빨간색 영역(위험)에 신규 데이터가 추가될 경우, 테두리 애니메이션(깜빡임) 효과나 폰트 강조를 적용하여 운영자의 즉각적인 대응 유도.

반응형 레이아웃:

Flexbox 또는 Grid 레이아웃을 사용하여 접속하는 디스플레이 크기에 맞춰 카드 크기가 자동으로 조정되도록 설계.

4. 기술 스택 및 환경
프론트엔드: HTML5, CSS3, Vanilla JavaScript (Vite 미사용).

백엔드/DB: Supabase (Realtime SDK 사용).
